## [1.0.0] - 2024-01-01

### 新機能

-   マルチログレベルの主要なロギング機能（Trace、Debug、Info、Warn、Error、Fatal、Panic）
-   オブジェクトプール付きのスレッドセーフロガーの実装
-   ビルドタグのサポート（デフォルト、デバッグ、リリース、ドロップモード）
-   デフォルトのテキストフォーマッター付きのカスタムフォーマッターインターフェース
-   マルチライタ出力のサポート
-   高スループットシナリオ向けの非同期書き込み機能
-   1時間ごとのログファイルローテーション
-   Goroutine ID とトレース ID トレース付きのコンテキスト感知ロギング
-   設定可能なスタック深さ付きのコーラー情報
-   グローバルパッケージレベルの簡易関数
-   Zap ロガー統合サポート

### パフォーマンス

-   `sync.Pool` を使用したエントリオブジェクトとバッファのオブジェクトプール化
-   高価な操作を避けるための早期レベルチェック
-   ブロッキングなしの非同期書き込み用の非同期ライタ
-   異なる環境向けのビルドタグによる最適化

### ビルドタグ

-   **デフォルト**: 完全な機能とデバッグメッセージ
-   **デバッグ**: 増強されたデバッグ情報とコーラー詳細
-   **リリース**: 本番環境向けの最適化、デバッグメッセージの無効化
-   **ドロップ**: 最大パフォーマンス、何も記録しない

### コア機能

-   **Logger**: 設定可能なレベル、出力、フォーマッターを備えた主なロガー構造
-   **Entry**: 包括的なメタデータ付きのログレコード構造
-   **Levels**: Panic（最高）から Trace（最低）の7つのログレベル
-   **Formatters**: プラグ可能なフォーマットシステム
-   **Writers**: ファイルローテーションと非同期書き込みのサポート
-   **Context**: Goroutine ID と分散トレースのサポート

### API ハイライト

-   メソッドチェーンを使用した流れるような設定 API
-   シンプルとフォーマットされたログメソッド（`.Info()` と `.Infof()`）
-   設定を分離するためのロガークローン
-   コンテキスト感知ロギング付き `CloneToCtx()`
-   プレフィックスとサフィックスメッセージのカスタマイズ
-   コーラー情報の切り替え

### テスト

-   カバレッジ率 93.5% の包括的なテストスイート
-   マルチビルドタグテストサポート
-   自動化されたテストワークフロー
-   パフォーマンスベンチマーク

## 移行ガイド

### v0.9.x から v1.0.0 への移行

#### 破壊的変更

-   なし - v1.0.0 は v0.9.x と後方互換

#### 利用可能な新機能

-   強化されたビルドタグサポート
-   包括的なドキュメント
-   専門的なプロジェクトテンプレート
-   セキュリティレポートプロセス

#### 推奨される更新

```go
// 旧方式（まだサポート）
logger := log.New()
logger.SetLevel(log.InfoLevel)

// 推奨される新しい方法、メソッドチェーンを使用
logger := log.New().
    SetLevel(log.InfoLevel).
    Caller(true).
    SetPrefixMsg("[MyApp] ")
```

### v0.8.x から v0.9.x への移行

#### 破壊的変更

-   非推奨の設定メソッドの削除
-   内部バッファ管理の変更

#### 移行手順

1. 必要に応じてインポートパスを更新
2. 非推奨のメソッドを置き換えます：

    ```go
    // 旧方式（非推奨）
    logger.SetOutputFile("app.log")

    // 新しい方式
    file, _ := os.Create("app.log")
    logger.SetOutput(file)
    ```

### v0.5.x およびそれ以前のバージョンからの移行

#### 主な変更

-   一貫性のために完全に再設計された API
-   オブジェクトプールを通じたパフォーマンスの強化
-   新しいビルドタグシステム

#### 移行が必要

-   すべてのロギング呼び出しを新しい API に更新
-   フォーマッター実装のレビューと更新
-   新しいビルドタグ設定を使用したテスト
