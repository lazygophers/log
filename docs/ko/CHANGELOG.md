## [1.0.0] - 2024-01-01

### 새로운 기능

-   다중 로그 레벨을 갖춘 핵심 로깅 기능(Trace, Debug, Info, Warn, Error, Fatal, Panic)
-   객체 풀을 갖춘 스레드 안전 로거 구현
-   빌드 태그 지원(기본값, 디버그, 릴리스, 폐기 모드)
-   기본 텍스트 포맷터를 갖춘 사용자 정의 포맷터 인터페이스
-   다중 라이터 출력 지원
-   높은 처리량 시나리오를 위한 비동기 쓰기 기능
-   자동 시간별 로그 파일 로테이션
-   Goroutine ID 및 추적 ID 추적이 포함된 컨텍스트 인식 로깅
-   구성 가능한 스택 깊이를 갖춘 호출자 정보
-   전역 패키지 수준의 편의 함수
-   Zap 로거 통합 지원

### 성능

-   엔트리 개체 및 버퍼의 객체 풀링을 위해 `sync.Pool` 사용
-   비용이 많이 드는 작업을 피하기 위한 조기 레벨 확인
-   차단 없는 로그 쓰기를 위한 비동기 라이터
-   다양한 환경을 위한 빌드 태그 최적화

### 빌드 태그

-   **기본값**: 디버그 메시지가 포함된 완전한 기능
-   **디버그**: 향상된 디버그 정보 및 호출자 세부 정보
-   **릴리스**: 프로덕션 최적화, 디버그 메시지 비활성화
-   **폐기**: 최대 성능, 작동하지 않는 로그 작업

### 핵심 기능

-   **Logger**: 구성 가능한 레벨, 출력, 포맷터를 갖춘 주요 로거 구조체
-   **Entry**: 포괄적인 메타데이터를 갖춘 로그 레코드 구조체
-   **Levels**: Panic(최상위)에서 Trace(최하위)까지의 7가지 로그 레벨
-   **Formatters**: 플러그 가능한 포맷 시스템
-   **Writers**: 파일 로테이션 및 비동기 쓰기 지원
-   **Context**: Goroutine ID 및 분산 추적 지원

### API 하이라이트

-   메서드 체이닝이 포함된 유창한 구성 API
-   간단하고 형식화된 로그 메서드(`.Info()` 및 `.Infof()`)
-   로거 복제를 통한 구성 격리
-   `CloneToCtx()`를 통한 컨텍스트 인식 로깅
-   접두사 및 접미사 메시지 사용자 정의
-   호출자 정보 전환

### 테스트

-  覆盖率 93.5%의 포괄적인 테스트 스위트
-   다중 빌드 태그 테스트 지원
-   자동화된 테스트 워크플로우
-   성능 벤치마크

## 마이그레이션 가이드

### v0.9.x에서 v1.0.0으로 마이그레이션

#### 주요 변경 사항

-   없음 - v1.0.0은 v0.9.x와 호환됩니다

#### 새로 사용 가능한 기능

-   향상된 빌드 태그 지원
-   포괄적인 문서
-   전문적인 프로젝트 템플릿
-   보안 보고 프로세스

#### 권장 업데이트

```go
// 이전 방법(여전히 지원됨)
logger := log.New()
logger.SetLevel(log.InfoLevel)

// 권장되는 새로운 방법, 메서드 체이닝 사용
logger := log.New().
    SetLevel(log.InfoLevel).
    Caller(true).
    SetPrefixMsg("[MyApp] ")
```

### v0.8.x에서 v0.9.x로 마이그레이션

#### 주요 변경 사항

-   사용 중단된 구성 메서드 제거
-   내부 버퍼 관리 변경

#### 마이그레이션 단계

1. 필요한 경우 가져오기 경로 업데이트
2. 사용 중단된 메서드 교체:

    ```go
    // 이전(사용 중단됨)
    logger.SetOutputFile("app.log")

    // 새로운
    file, _ := os.Create("app.log")
    logger.SetOutput(file)
    ```

### v0.5.x 이전에서 마이그레이션

#### 주요 변경 사항

-   더 나은 일관성을 위한 API의 완전한 재설계
-   객체 풀을 통한 성능 강화
-   새로운 빌드 태그 시스템

#### 마이그레이션 필요

-   새 API로 모든 로그 호출 업데이트
-   포맷터 구현 검토 및 업데이트
-   새 빌드 태그 구성으로 테스트
