## ✨ Особенности

-   **🚀 Высокая производительность**: Построена на zap с повторным использованием объектов Entry через пул, снижая выделение памяти
-   **📊 Богатые уровни логирования**: Уровни Trace, Debug, Info, Warn, Error, Fatal, Panic
-   **⚙️ Гибкая конфигурация**:
    -   Контроль уровня логирования
    -   Запись информации о вызывающем
    -   Информация трассировки (включая ID горутины)
    -   Пользовательские префиксы и суффиксы логов
    -   Пользовательские целевые выходы (консоль, файлы и т.д.)
    -   Опции форматирования логов
-   **🔄 Ротация файлов**: Поддержка ежечасной ротации лог-файлов
-   **🔌 Совместимость с Zap**: Бесшовная интеграция с zap WriteSyncer
-   **🎯 Простой API**: Чистый API, похожий на стандартную библиотеку логирования, легко использовать

## 🚀 Быстрый старт

### Установка

:::tip Установка
```bash
go get github.com/lazygophers/log
```
:::

### Базовое использование

```go title="Быстрый старт"
package main

import (
    "github.com/lazygophers/log"
)

func main() {
    // Использовать глобальный logger по умолчанию
    log.Debug("Отладочная информация")
    log.Info("Общая информация")
    log.Warn("Предупреждение")
    log.Error("Ошибка")

    // Использовать форматированный вывод
    log.Infof("Пользователь %s вошел в систему", "admin")

    // Пользовательская конфигурация
    customLogger := log.New().
        SetLevel(log.InfoLevel).
        EnableCaller(false).
        SetPrefixMsg("[MyApp]")

    customLogger.Info("Это сообщение от custom logger")
}
```

## 📚 Расширенное использование

### Logger с файловым выводом

```go title="Конфигурация файлового вывода"
package main

import (
    "os"
    "github.com/lazygophers/log"
)

func main() {
    // Создать logger с файловым выводом
    logger := log.New().
        SetLevel(log.DebugLevel).
        EnableCaller(true).
        EnableTrace(true).
        SetOutput(os.Stdout, log.GetOutputWriterHourly("/var/log/myapp.log"))

    logger.Debug("Отладочное сообщение с информацией о вызывающем")
    logger.Info("Общее сообщение с информацией трассировки")
}
```

### Контроль уровней логирования

```go title="Контроль уровней логирования"
package main

import "github.com/lazygophers/log"

func main() {
    logger := log.New().SetLevel(log.WarnLevel)

    // Только сообщения warn и выше будут записываться
    logger.Debug("Это не будет записано")  // Игнорируется
    logger.Info("Это не будет записано")   // Игнорируется
    logger.Warn("Это будет записано")    // Записывается
    logger.Error("Это будет записано")   // Записывается
}
```

## 🎯 Сценарии использования

### Подходящие сценарии

-   **Web-сервисы и API бэкенд**: Трассировка запросов, структурированные логи, мониторинг производительности
-   **Архитектура микросервисов**: Распределенная трассировка (TraceID), единый формат логов, высокая пропускная способность
-   **Инструменты командной строки**: Контроль уровней,简洁ный вывод, отчеты об ошибках
-   **Пакетные задачи**: Ротация файлов, длительное выполнение, оптимизация ресурсов

### Особые преимущества

-   **Оптимизация через пул объектов**: Повторное использование объектов Entry и Buffer для снижения давления GC
-   **Асинхронная запись**: Сценарии с высокой пропускной способностью (10000+ записей/сек) без блокировки
-   **Поддержка TraceID**: Трассировка запросов распределенных систем, интеграция с OpenTelemetry
-   **Запуск без конфигурации**: Готов к использованию,渐进ная конфигурация

## 🔧 Опции конфигурации

:::note Опции конфигурации
Все методы поддерживают цепочки вызовов, могут быть объединены для построения custom Logger.
:::

### Конфигурация Logger

| Метод                  | Описание                         | Значение по умолчанию |
| ----------------------- | -------------------------------- | -------------------- |
| `SetLevel(level)`       | Установить минимальный уровень логирования | `DebugLevel`         |
| `EnableCaller(enable)`  | Включить/отключить запись информации о вызывающем | `false`      |
| `EnableTrace(enable)`   | Включить/отключить информацию трассировки | `false`      |
| `SetCallerDepth(depth)` | Установить глубину вызывающего   | `2`          |
| `SetPrefixMsg(prefix)`  | Установить префикс лога         | `""`         |
| `SetSuffixMsg(suffix)`  | Установить суффикс лога         | `""`         |
| `SetOutput(writers...)` | Установить цели вывода          | `os.Stdout`  |

### Уровни логирования

| Уровень       | Описание                                  |
| ------------- | ----------------------------------------- |
| `TraceLevel`  | Самый подробный, для детальной трассировки |
| `DebugLevel`  | Отладочная информация                      |
| `InfoLevel`   | Информационные сообщения                  |
| `WarnLevel`   | Предупреждающие сообщения                  |
| `ErrorLevel`  | Сообщения об ошибках                      |
| `FatalLevel`  | Критические ошибки (вызывает os.Exit(1)) |
| `PanicLevel`  | Панические ошибки (вызывает panic())       |

## 🏗️ Архитектура

### Основные компоненты

-   **Logger**: Основная структура логирования с настраиваемыми уровнем, выводом, форматизатором и глубиной вызывающего
-   **Entry**: Отдельная запись лога с полной поддержкой метаданных
-   **Level**: Определения уровней логирования и утилитарные функции
-   **Format**: Интерфейс и реализация форматирования логов

### Оптимизация производительности

-   **Пул объектов**: Повторное использование объектов Entry для уменьшения выделения памяти
-   **Условная запись**: Запись дорогих полей только при необходимости
-   **Быстрая проверка уровней**: Проверка уровней на самом внешнем уровне
-   **Безблокировочный дизайн**: Большинство операций не требуют блокировок

## ❓ Часто задаваемые вопросы

### Как выбрать подходящий уровень логирования?

-   **Среда разработки**: Использовать `DebugLevel` или `TraceLevel` для получения подробной информации
-   **Производственная среда**: Использовать `InfoLevel` или `WarnLevel` для снижения нагрузки
-   **Тестирование производительности**: Использовать `PanicLevel` для отключения всех логов

### Как оптимизировать производительность в продакшене?

:::warning Внимание
В сценариях с высокой пропускной способностью рекомендуется использовать асинхронную запись и разумные уровни логирования для оптимизации производительности.
:::

1. Использовать `AsyncWriter` для асинхронной записи:

```go title="Конфигурация асинхронной записи"
writer := log.GetOutputWriterHourly("./logs/app.log")
asyncWriter := log.NewAsyncWriter(writer, 5000)
logger.SetOutput(asyncWriter)
```

2. Настроить уровни логирования, чтобы избежать ненужных логов:

```go title="Оптимизация уровней"
logger.SetLevel(log.InfoLevel)  // Пропустить Debug и Trace
```

3. Использовать условные логи для снижения нагрузки:

```go title="Условные логи"
if logger.Level >= log.DebugLevel {
    logger.Debug("Подробная отладочная информация")
}
```

### В чем разница между `Caller` и `EnableCaller`?

-   **`EnableCaller(enable bool)`**: Управляет сбором информации о вызывающем в Logger
    -   `EnableCaller(true)` включает сбор информации о вызывающем
-   **`Caller(disable bool)`**: Управляет отображением информации о вызывающем в Formatter
    -   `Caller(true)` отключает отображение информации о вызывающем

Рекомендуется использовать `EnableCaller` для глобального управления.

### Как реализовать custom форматизатор?

Реализовать интерфейс `Format`:

```go title="Custom форматизатор"
type MyFormatter struct{}

func (f *MyFormatter) Format(entry *log.Entry) []byte {
    return []byte(fmt.Sprintf("[%s] %s\n",
        entry.Level.String(), entry.Message))
}

logger.SetFormatter(&MyFormatter{})
```
