## ✨ Возможности

-   **🚀 Высокая производительность**: Построена на базе zap с использованием пула объектов и условной записью полей
-   **📊 Разнообразные уровни логирования**: Уровни Trace, Debug, Info, Warn, Error, Fatal, Panic
-   **⚙️ Гибкая конфигурация**:
    -   Управление уровнем логирования
    -   Запись информации о вызове
    -   Информация о трассировке (включая ID горутины)
    -   Пользовательские префиксы и суффиксы логов
    -   Пользовательские цели вывода (консоль, файлы и т.д.)
    -   Опции форматирования логов
-   **🔄 Вращение файлов**: Поддержка вращения файлов логов каждый час
-   **🔌 Совместимость с Zap**: Легкая интеграция с zap WriteSyncer
-   **🎯 Простой API**: Чистый API, похожий на стандартную библиотеку логирования, прост в использовании

## 🚀 Быстрый старт

### Установка

:::tip Установка
```bash
go get github.com/lazygophers/log
```
:::

### Базовое использование

```go title="Быстрый старт"
package main

import (
    "github.com/lazygophers/log"
)

func main() {
    // Использование глобального logger по умолчанию
    log.Debug("Отладочное сообщение")
    log.Info("Информационное сообщение")
    log.Warn("Предупреждающее сообщение")
    log.Error("Сообщение об ошибке")

    // Использование форматированного вывода
    log.Infof("Пользователь %s успешно вошел в систему", "admin")

    // Пользовательская конфигурация
    customLogger := log.New().
        SetLevel(log.InfoLevel).
        EnableCaller(false).
        SetPrefixMsg("[МоеПриложение]")

    customLogger.Info("Это сообщение из пользовательского logger")
}
```

## 📚 Расширенное использование

### Пользовательский logger с выводом в файл

```go title="Конфигурация вывода в файл"
package main

import (
    "os"
    "github.com/lazygophers/log"
)

func main() {
    // Создание logger с выводом в файл
    logger := log.New().
        SetLevel(log.DebugLevel).
        EnableCaller(true).
        EnableTrace(true).
        SetOutput(os.Stdout, log.GetOutputWriterHourly("/var/log/myapp.log"))

    logger.Debug("Отладочное сообщение с информацией о вызове")
    logger.Info("Информационное сообщение с информацией о трассировке")
}
```

### Управление уровнем логирования

```go title="Управление уровнем логирования"
package main

import "github.com/lazygophers/log"

func main() {
    logger := log.New().SetLevel(log.WarnLevel)

    // Будут записаны только уровни warn и выше
    logger.Debug("Это не будет записано")  // Пропущено
    logger.Info("Это не будет записано")   // Пропущено
    logger.Warn("Это будет записано")    // Записано
    logger.Error("Это будет записано")   // Записано
}
```

## 🎯 Сценарии использования

### Применимые сценарии

-   **Веб-сервисы и бэкенд API**: Отслеживание запросов, структурированное логирование, мониторинг производительности
-   **Микросервисная архитектура**: Распределенная трассировка (TraceID), унифицированный формат логов, высокая пропускная способность
-   **Командные инструменты**: Управление уровнями, чистый вывод, отчеты об ошибках
-   **Пакетные задачи**: Вращение файлов, длительное выполнение, оптимизация ресурсов

### Особые преимущества

-   **Оптимизация с пулом объектов**: Переиспользование объектов Entry и Buffer, снижение нагрузки на GC
-   **Асинхронная запись**: Высокая пропускная способность (10000+ записей/сек) без блокировок
-   **Поддержка TraceID**: Отслеживание запросов в распределенных системах, интеграция с OpenTelemetry
-   **Запуск без конфигурации**: Готов к использованию, постепенная настройка

## 🔧 Опции конфигурации

:::note Опции конфигурации
Все следующие методы поддерживают цепочное вызывание и могут быть скомбинированы для создания пользовательского Logger.
:::

### Конфигурация Logger

| Метод                  | Описание                | Значение по умолчанию       |
| --------------------- | ------------------- | ----------- |
| `SetLevel(level)`       | Установить минимальный уровень логирования     | `DebugLevel` |
| `EnableCaller(enable)`  | Включить/отключить информацию о вызове  | `false`      |
| `EnableTrace(enable)`   | Включить/отключить информацию о трассировке    | `false`      |
| `SetCallerDepth(depth)` | Установить глубину вызова       | `2`          |
| `SetPrefixMsg(prefix)`  | Установить префикс лога         | `""`         |
| `SetSuffixMsg(suffix)`  | Установить суффикс лога         | `""`         |
| `SetOutput(writers...)` | Установить цели вывода         | `os.Stdout`  |

### Уровни логирования

| Уровень        | Описание                        |
| ----------- | --------------------------- |
| `TraceLevel` | Самый подробный, для детальной трассировки        |
| `DebugLevel` | Информация для отладки                    |
| `InfoLevel`  | Общая информация                    |
| `WarnLevel`  | Предупреждающие сообщения                    |
| `ErrorLevel` | Сообщения об ошибках                    |
| `FatalLevel` | Фатальные ошибки (вызывает os.Exit(1))    |
| `PanicLevel` | Ошибки паники (вызывает panic())    |

## ❓ Часто задаваемые вопросы

### Как выбрать подходящий уровень логирования?

-   **Среда разработки**：Использовать `DebugLevel` или `TraceLevel` для получения подробной информации
-   **Среда производства**：Использовать `InfoLevel` или `WarnLevel` для снижения накладных расходов
-   **Тестирование производительности**：Использовать `PanicLevel` для отключения всех логов

### Как оптимизировать производительность в производственной среде?

:::warning Примечание
В сценариях с высокой пропускной способностью рекомендуется использовать асинхронную запись с разумными уровнями логирования для оптимизации производительности.
:::

1. Использовать `AsyncWriter` для асинхронной записи：

```go title="Конфигурация асинхронной записи"
writer := log.GetOutputWriterHourly("./logs/app.log")
asyncWriter := log.NewAsyncWriter(writer, 5000)
logger.SetOutput(asyncWriter)
```

2. Настроить уровень логирования для избежания ненужных логов：

```go title="Оптимизация уровня"
logger.SetLevel(log.InfoLevel)  // Пропустить Debug и Trace
```

3. Использовать условные логи для снижения затрат：

```go title="Условные логи"
if logger.Level >= log.DebugLevel {
    logger.Debug("Подробная отладочная информация")
}
```

### В чем разница между `Caller` и `EnableCaller`?

-   **`EnableCaller(enable bool)`**：Контролирует, собирает ли Logger информацию о вызове
    -   `EnableCaller(true)` включает сбор информации о вызове
-   **`Caller(disable bool)`**：Контролирует, отображает ли Formatter информацию о вызове
    -   `Caller(true)` отключает вывод информации о вызове

Рекомендуется использовать `EnableCaller` для глобального управления.

### Как реализовать пользовательский форматтер?

Реализуйте интерфейс `Format`：

```go title="Пользовательский форматтер"
type MyFormatter struct{}

func (f *MyFormatter) Format(entry *log.Entry) []byte {
    return []byte(fmt.Sprintf("[%s] %s\n",
        entry.Level.String(), entry.Message))
}

logger.SetFormatter(&MyFormatter{})
```
