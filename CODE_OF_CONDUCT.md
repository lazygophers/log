# 📋 行为准则

## 我们的承诺

作为成员、贡献者和领导者，我们承诺为每个人创造一个无骚扰的社区参与体验，无论年龄、体型、可见或不可见的残疾、种族、性别特征、性别认同和表达、经验水平、教育、社会经济地位、国籍、个人外貌、种族、种姓、肤色、宗教或性取向和身份如何。

我们承诺以促进开放、欢迎、多元、包容和健康社区的方式行动和互动。

## 我们的标准

为社区创造积极环境的行为示例包括：

### ✅ 鼓励的行为

-   **对他人表现出同理心和善意**
-   **尊重不同的意见、观点和经验**
-   **给予并优雅接受建设性反馈**
-   **承担责任**并向受我们错误影响的人道歉，并从经验中学习
-   **关注对整体社区最有利的事情**，而不仅仅是个人的利益
-   **使用欢迎和包容的语言**
-   **尊重不同的意见、观点和经验**
-   **对贡献表示感谢**，无论贡献大小
-   **帮助新人**入门并感到受欢迎

### ❌ 不可接受的行为

不可接受的行为示例包括：

-   **骚扰**：使用性暗示语言或图像，以及任何形式的性关注或接近
-   **挑衅**：侮辱性或贬低性评论，以及个人或政治攻击
-   **歧视**：针对任何受保护群体的歧视性言论或行为
-   **隐私侵犯**：未经明确许可发布他人的私人信息，如物理地址或电子邮件地址
-   **垃圾信息**：过度或无关的发布，或商业广告
-   **无关内容**：持续发布与项目无关的内容
-   **其他行为**：在专业环境中可能被认为不合适的其他行为

## 执行责任

社区领导者负责澄清和执行我们的可接受行为标准，并将对任何他们认为不适当、威胁性、冒犯性或有害的行为采取适当和公平的纠正措施。

社区领导者有权并有责任删除、编辑或拒绝不符合本行为准则的评论、提交、代码、维基编辑、问题和其他贡献，并在适当时沟通审核决策的原因。

## 适用范围

本行为准则适用于所有社区空间，也适用于个人在公共空间中正式代表社区时。代表我们社区的示例包括使用官方电子邮件地址、通过官方社交媒体账户发布内容，或在线上或线下活动中担任指定代表。

### 社区空间包括

-   **GitHub 仓库**及所有相关空间（问题、拉取请求、讨论、维基）
-   **官方沟通渠道**（Discord、Slack、邮件列表）
-   **代表项目的社交媒体**账户和帖子
-   **项目参与的活动**（会议、聚会、工作坊）
-   **文档**和其他官方项目材料

## 执行机制

### 报告

滥用、骚扰或其他不可接受行为的实例可以向负责执行的社区领导者报告：

-   **主要联系人**：support@lazygophers.com
-   **GitHub 问题**：对于公开违规行为，您可以通过 GitHub 问题报告
-   **私人报告**：对于敏感事项，请直接通过电子邮件联系我们

所有报告都将及时、公平地审查和调查。所有社区领导者都有义务尊重任何事件报告者的隐私和安全。

### 响应时间表

-   **确认**：24-48 小时内
-   **初步响应**：72 小时内
-   **解决**：1-2 周内（复杂案件可能需要更长时间）

## 执行指南

社区领导者将遵循这些社区影响指南来确定他们认为违反本行为准则的任何行为的后果：

### 1. 纠正

**社区影响**：使用不适当的语言或其他被认为在社区中不专业或不受欢迎的行为。

**后果**：社区领导者发出的私人书面警告，明确违反行为的性质和解释为什么该行为不适当。可能会要求公开道歉。

**示例行动**：

-   解释问题的私人消息
-   要求编辑或删除有问题的内容
-   暂时限制访问特定项目区域

### 2. 警告

**社区影响**：通过单个事件或一系列行动违反准则。

**后果**：带有持续行为后果的警告。在规定时间内不得与相关人员互动，包括与执行行为准则的人员进行未经请求的互动。这包括避免在社区空间以及社交媒体等外部渠道的互动。违反这些条款可能导致暂时或永久禁令。

**示例行动**：

-   带有具体行为期望的书面警告
-   暂时限制社区参与
-   监控未来贡献

### 3. 暂时禁令

**社区影响**：严重违反社区标准，包括持续的不当行为。

**后果**：在规定时间内暂时禁止与社区进行任何形式的互动或公开沟通。在此期间不允许与相关人员（包括与执行行为准则的人员）进行公开或私人互动。违反这些条款可能导致永久禁令。

**持续时间**：通常为 1-3 个月，取决于严重程度

### 4. 永久禁令

**社区影响**：表现出违反社区标准的模式，包括持续的不当行为、对个人的骚扰，或对群体进行攻击或贬低。

**后果**：永久禁止在社区内进行任何形式的公开互动。

## 申诉流程

如果您认为受到了错误的制裁，您可以通过以下方式申诉：

1. **发送电子邮件**至support@lazygophers.com，在 30 天内联系我们团队
2. **提供**详细的申诉理由
3. **包括**任何相关证据或背景信息
4. **等待**由不同的社区领导者组进行审查

申诉将在 2-3 周内审查，决定将是最终的。

## 积极社区建设

我们鼓励社区成员：

### 🤝 促进包容

-   **用有用的指导欢迎新人**
-   **庆祝背景和观点的多样性**
-   **在所有沟通中使用包容性语言**
-   **对学习或提问的人保持耐心**

### 🎯 关注解决方案

-   **提供建设性反馈**而不仅仅是批评
-   **在看到有人遇到困难时提供帮助**
-   **慷慨地分享知识和资源**
-   **合作而非竞争**

### 🌟 认可贡献

-   **感谢贡献者的工作**，无论多么微小
-   **突出社区成员的成就**
-   **指导正在学习的他人**
-   **适当地认可他人的想法和工作**

## 技术标准

除了行为标准外，我们还维护技术标准：

### 代码质量

-   遵循既定的编码规范
-   编写清晰、文档完善的代码
-   彻底测试您的贡献
-   对代码审查反馈持开放态度

### 沟通

-   保持讨论与项目相关
-   在问题和 PR 中使用清晰、专业的语言
-   报告错误或请求功能时提供有用的背景信息
-   在等待维护者回复时保持耐心
